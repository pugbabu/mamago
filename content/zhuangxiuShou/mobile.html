# 预发分支

  • 分支： advance

接到新的需求，首先从dev分支上拉一个分支，在新拉的分支上进行开发

git checkout -b yourbranchName
1

当功能开发完成，需要提交测试时，打包本地分支代码，发布到测试服务器上。如果遇到多个同事同时在测试环境测试，可以商量新建一个分支，合并到新建分支，用新建的分支代码打包测试。本地测完后，打包到预发服务器上继续测试。最终测试通过且产品验收完毕，需要和慧青说一声，慧青确认后，pull dev代码到你自己分支，解决冲突后，合并到dev，最后由慧青发布到正式服务器。
